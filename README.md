📊 Business Intelligence: Análise Estratégica de Vendas e Lucratividade

O Projeto
O objetivo desta análise foi transformar uma base de dados de vendas bruta em um cockpit de decisão. A ideia central foi ir além do faturamento (vendas) e focar na saúde financeira da operação, identificando onde o volume de vendas não está se traduzindo em lucro real.

Ferramenta Utilizada

Tableau: Utilizado para a conexão, modelagem e visualização dos dados.

Cálculos Avançados: Implementação de campos calculados (LODs e Funções de Tabela) para métricas como Margem de Lucro, Crescimento MoM% e SLA de Entrega.

O que aprendi no processo
Durante o desenvolvimento, aprofundei meus conhecimentos em:

UX para Dashboards: Como usar cores de forma semântica (alertas críticos em vermelho) para guiar o olho do gestor para o que realmente importa.

Parâmetros Dinâmicos: Criação de controles de granularidade temporal (dia/semana/mês) para permitir análises macro e micro em um único gráfico.

Lógica de Negócio aplicada ao BI: A importância de cruzar vendas vs. lucro para identificar subcategorias que, apesar de populares, operam com margens negativas.

Os Resultados
O dashboard final permite identificar instantaneamente:

Zonas de Risco: Estados e subcategorias que estão operando no prejuízo (destacados pelo Loss Alert).

Tendências de Mercado: Sazonalidade de vendas ao longo dos anos com comparação direta de performance.

Eficiência Operacional: Visão clara de quais produtos impulsionam o lucro e quais precisam de revisão estratégica de preços ou custos logísticos.

Durante a implementação da data dinâmica, superei desafios de sintaxe no Tableau utilizando blocos CASE/WHEN para garantir que o parâmetro de granularidade interagisse corretamente com as funções de data do motor de cálculo.
